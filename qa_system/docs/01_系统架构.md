# 系统架构文档

## 1. 整体架构

本系统为**知识问答子系统**，基于 LangChain + Neo4j + MySQL + 大语言模型构建，为海外藏中国文物提供自然语言问答服务。

## 2. 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| Web 框架 | FastAPI | ASGI 高性能框架，支持 WebSocket |
| LLM 框架 | LangChain 1.3.2 | `langchain.agents.create_agent` 基于 LangGraph |
| LLM 客户端 | `langchain_openai.ChatOpenAI` | OpenAI 兼容协议，base_url 可配置 |
| 图数据库 | Neo4j | Cypher 查询，当前处于 off 状态 |
| 关系数据库 | MySQL (aiomysql) | artifact 表 + ai_history 表 |
| 包管理 | uv | Python 虚拟环境和依赖管理 |

## 3. 配置开关

通过 `.env` 文件控制功能开关：

```bash
ENABLE_MYSQL=on    # 启用 MySQL Tool（当前 always on）
ENABLE_NEO4J=off    # Neo4j Tool（待连接恢复后开启）
```

## 4. 核心模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| 配置 | `config.py` | 统一读取 .env，提供 Settings 单例 |
| HTTP 路由 | `app/api/qa_router.py` | REST 接口：session/ask/history/feedback |
| WS 路由 | `app/api/ws_router.py` | WebSocket 端点：/api/qa/ws |
| 会话管理 | `app/core/session_manager.py` | 创建会话、存储历史、流式写入、cursor 查询 |
| WS 管理 | `app/core/ws_manager.py` | session→client 映射、广播、断线重连 |
| 流式 Agent | `app/core/streaming_agent.py` | StreamingChatOpenAI 劫持 LLM token 流 |
| Graph Agent | `app/agents/graph_agent.py` | 意图识别 + Tool 调用，基于 LangGraph |
| MySQL Tool | `app/agents/tools/mysql_tool.py` | 9 个 LangChain Tool，接真实 artifact 表 |
| Neo4j Tool | `app/agents/tools/neo4j_tool.py` | 3 个 LangChain Tool（当前 disabled） |

| LLM 生成器 | `app/retrieval/llm_generator.py` | ChatOpenAI 工厂函数，支持任意 OpenAI 兼容服务 |

## 5. 数据流

### HTTP 问答流程

```
用户提问 → POST /api/qa/ask
  ├─ 有 session_id → 从 ai_history 加载历史 → inject 到 Graph Agent
  ├─ 无 session_id → 从零开始单轮问答
  └─ Graph Agent 调用 Tool → DB 查询 → 结果转自然语言
      → AnswerBuilder 润色 → 返回 AskResponse
```

### WebSocket 流式流程

```
WS 连接 /api/qa/ws?session_id=xxx
  ├─ 连接确认 + 断点检测
  │   ├─ streaming_done=TRUE → 正常
  │   └─ streaming_done=FALSE → 立即推送 resume_remaining
  │
  ├─ 客户端发送 message 事件
  │   └─ process_streaming() 异步执行
  │       ├─ 保存 user 消息到 ai_history
  │       ├─ 保存空的 assistant 消息（streaming_done=FALSE）
  │       ├─ stream_agent_tokens() 迭代 LLM token
  │       │   ├─ 每个 token → WS.send_chunk() 推送客户端
  │       │   └─ 每 20 字符批量写入 ai_history
  │       └─ 流式结束 → WS.send_done() + update_stream_done()
  │
  └─ 客户端断线重连 → 带 cursor 续写
      └─ resume_stream() 从 offset 处推送剩余内容
```

## 6. 流式实现原理

LangGraph 的 `agent.astream()` 本身是异步迭代器，每次 `yield` 返回整个图的 state snapshot。

通过 `StreamingChatOpenAI` 子类劫持底层 `_stream()` 方法，将每个 token 写入队列；后台协程消费队列并实时推送给 WebSocket 客户端，从而实现 token 级别的流式输出。
