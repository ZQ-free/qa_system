## 2. artifact 表（文物详情）

共 **6997 条记录**，35 个字段，存储海外藏中国文物的完整结构化信息。

### 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `object_id` | VARCHAR(255) | **主键**，全球唯一标识（如 `ld1-1643381040022-1643381041048-0`） |
| `artifact_id` | VARCHAR(256) | 实体ID，格式 `entity:artifact:N:id`，有索引 |
| `museum_id` | INT | 博物馆ID |

| `artist_province` | VARCHAR(100) | 艺术家省份 |
| `dynasty` | VARCHAR(200) | 朝代（中英文混合，如 `Qing（清）`、`Tang（唐）`） |
| `artist_wikidata_id` | VARCHAR(32) | 艺术家 Wikidata ID |
| `artist_birth` | VARCHAR(120) | 艺术家出生年份 |
| `artist_death` | VARCHAR(120) | 艺术家去世年份 |
| `artist_bio` | VARCHAR(4000) | 艺术家简介 |
| `artist_wikipedia_summary` | VARCHAR(4000) | 维基百科摘要 |
| `artist_enriched_at` | VARCHAR(32) | 数据富化时间 |
| `period` | VARCHAR(200) | 创作时期（如 `1890s`、`18th century`） |
| `period_start_year` | SMALLINT | 起始年份 |
| `period_end_year` | SMALLINT | 结束年份 |
| `type` | VARCHAR(100) | 文物类型（`Paintings`、`Ceramics`、`Sculptures`、`Bronzes` 等） |
| `material` | TEXT | 材质/媒介描述（如 `oil on canvas`、`porcelain`） |
| `culture` | VARCHAR(300) | 文化/主题标签 |
| `description` | TEXT | 描述文本 |
| `provenance` | TEXT | 出土/流传信息 |

| `image_count` | SMALLINT | 图片数量 |
| `credit_line` | TEXT | 藏品说明/捐赠信息 |
| `accession_number` | VARCHAR(200) | 登记号 |
| `crawl_date` | DATE | 爬取日期 |

### 索引

- `object_id`：主键
- `artifact_id`：有索引（MUL）

## 3. ai_history 表（对话历史与流式状态）

用于存储多轮对话历史，支持流式断点续写。

### 建表语句

```sql
CREATE TABLE IF NOT EXISTS ai_history (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    session_id     VARCHAR(36)  NOT NULL COMMENT '会话唯一标识，UUID',
    role            ENUM('user','assistant','system','tool') NOT NULL,
    content         TEXT         NOT NULL COMMENT '消息正文',
    tool_name       VARCHAR(128)  DEFAULT NULL COMMENT '调用工具名（仅 tool 角色）',
    tool_input      TEXT         DEFAULT NULL COMMENT '工具输入参数 JSON',
    tool_output     TEXT         DEFAULT NULL COMMENT '工具返回结果 JSON',
    intent          VARCHAR(64)  DEFAULT NULL COMMENT '识别的意图类型（仅 assistant）',
    entity          VARCHAR(256)  DEFAULT NULL COMMENT '抽取的实体（仅 assistant）',
    token_count     INT UNSIGNED  DEFAULT 0 COMMENT '总 token 数（流式结束后更新）',
    sent_offset     INT UNSIGNED  DEFAULT 0 COMMENT '已发送给客户端的字符偏移量',
    streaming_done  BOOLEAN       DEFAULT FALSE COMMENT '流式是否已完成',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_session_id (session_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT UNSIGNED | **主键，自增**，作为游标主键（`message_id` 概念） |
| `session_id` | VARCHAR(36) | 会话唯一标识，创建会话时生成 UUID |
| `role` | ENUM | `user` / `assistant` / `system` / `tool` |
| `content` | TEXT | 消息正文；流式进行中时逐步拼接 |
| `tool_name` | VARCHAR(128) | 仅 `tool` 角色时填充，工具名 |
| `tool_input` | TEXT | 仅 `tool` 角色时填充，JSON 序列化的输入参数 |
| `tool_output` | TEXT | 仅 `tool` 角色时填充，工具返回结果 |
| `intent` | VARCHAR(64) | 仅 `assistant` 角色时填充，识别的意图类型 |
| `entity` | VARCHAR(256) | 仅 `assistant` 角色时填充，抽取的实体 |
| `token_count` | INT UNSIGNED | 流式结束后填入消耗的 token 总数 |
| `sent_offset` | INT UNSIGNED | **流式续写关键字段**：已发送给客户端的字符偏移量 |
| `streaming_done` | BOOLEAN | **流式续写关键字段**：`FALSE` = 正在生成中，`TRUE` = 已完成 |
| `created_at` | DATETIME | 创建时间 |

### 索引

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| PRIMARY | `id` | 主键 | 精确查询 |
| `idx_session_id` | `session_id` | 普通索引 | 按会话查历史 |
| `idx_created_at` | `created_at` | 普通索引 | 时间排序 |

### 流式写入流程

```
1. 用户发送消息 → 保存 role=user 的记录
2. AI 开始生成 → 保存 role=assistant 的记录（content='', streaming_done=FALSE）
3. LLM 输出 token → 每个 token:
   ├─ 推送 WS 客户端
   ├─ 累积到 20 字符 → UPDATE ai_history SET content = CONCAT(content, chunk)
   └─ sent_offset 自动更新为 LENGTH(content)
4. 流式结束:
   ├─ UPDATE ai_history SET streaming_done=TRUE, token_count=xxx, sent_offset=LENGTH(content)
   └─ WS 发送 done 事件，附 cursor = "id+LENGTH(content)"
```

### 断点续写原理

```
客户端断线时：streaming_done=FALSE，sent_offset=1520，content="已发送给客户端的内容"

客户端重连时：
1. 查 get_last_message(session_id)
2. 发现 streaming_done=FALSE
3. 发送 resume_remaining: remaining=content[sent_offset:]（未发送给客户端的部分）
4. 若 streaming_done=TRUE 且 cursor 被传入：从 cursor 指定的 offset 处继续
```

### Cursor 格式

```
"{message_id}+{sent_offset}"
例: "124+1520"  →  message_id=124，已发送 1520 字符，从第 1520 字符继续
```
